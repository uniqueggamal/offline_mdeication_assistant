# Offline Medication Assistant MVP Specification
## Overview
This document specifies the EXACT MVP built in the Flutter project at `c:/Users/Lenovo/Desktop/apps/offline_medication_assistant`. Any AI can recreate it identically by following these instructions. The app is an **offline-first medication reminder/assistant** with local SQLite storage, notifications, TTS, speech recognition, and optional Firebase sync. Architecture: Clean (services, data, features, UI).

**App Features (MVP):**
- Manage medications: Add/edit name, schedule times, groups, images.
- View medications by group/time.
- Log intakes (mark taken/missed).
- Voice/Text assistant: Parse intents (e.g., 'took my pill'), TTS responses.
- Caretaker mode: Switch patient ID.
- Logs screen.
- Home with bottom nav: Medications (primary), others stubbed.
- Offline-first, local user 'local_patient'.

**Tech Stack:**
- Flutter 3.x, Material 3, Teal theme.
- Platforms: Android/iOS/Web/Desktop (standard boilerplate).

## Exact lib/main.dart
```
import 'package:flutter/material.dart';
import 'core/services/app_services.dart';
import 'ui/screens/home_screen.dart';
import 'ui/widgets/app_scope.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final services = AppServices();
  await services.notifications.init();
  const userId = 'local_patient';  // MVP: offline-first
  runApp(
    AppScope(
      services: services,
      userId: userId,
      child: const OfflineMedicationAssistantApp(),
    ),
  );
}

class OfflineMedicationAssistantApp extends StatelessWidget {
  const OfflineMedicationAssistantApp({super.key});
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Offline Medication Assistant',
      theme: ThemeData(
        colorScheme: ColorScheme.fromSeed(seedColor: Colors.teal),
        useMaterial3: true,
      ),
      home: const HomeScreen(),
    );
  }
}
```

## Key Services (core/services/)
- **AppServices**: Orchestrates SqliteDb, NotificationService, TtsService, SpeechService, etc. Init notifications.
- **DbService**: Wraps SqliteDb for CRUD (medications, logs, users).
- **NotificationService**: FlutterLocalNotificationsPlugin for reminders.
- **TtsService**: FlutterTts for assistant responses.
- **SpeechService**: speech_to_text init (modelPath optional).
- **IntentEngine**: Parser + generator for assistant.
- **SyncService**: Firebase optional sync.

## Key UI/Screens:
- **HomeScreen**: Scaffold + BottomNavigationBar (MedicationsScreen primary).
- **MedicationsScreen**: MedicationViewController, groups (MedicationGrouping), MedicationCard widgets.
- **MedicationEditorScreen**: Form + image_picker preview.
- **AssistantScreen**: TextField + chips, parse intents, TTS response.
- **AppScope**: InheritedWidget provides services/userId.

## Implementation Notes
- All screens use StatefulWidget + controllers (ChangeNotifier).
- Search patterns: MedicationCard StatelessWidget, GroupSection, IntentParser normalizes input -> IntentResult.
- DB: 'offline_medication_assistant.db' SQLite.
